# PurpleForge（日本語）

**PurpleForge** は、Red Team（攻撃）、Blue Team（検知）、Purple Team（統合的視点）を組み合わせた、長期的かつ実践的な Purple Teaming リファレンスリポジトリです。

本リポジトリでは、資格情報の窃取、横移動、設定不備、検知回避などの攻撃手法を、攻撃側・防御側の両方の観点から体系的に文書化しています。

## 構成

本リポジトリは、環境構築、攻撃シミュレーション、自動化、検知エンジニアリング、インシデントレスポンスなど、Purple Teaming における主要な領域ごとに整理されています。

すべての内容は Markdown 形式で記載され、設定例、分析手順、必要に応じてビジュアル資料を含みます。  
各セクションは単体でも利用可能で、エンドツーエンドのワークフローとしても活用できます。

## 主なトピック

- フィッシングからランサムウェアまでのエンドツーエンド攻撃シナリオ  
- 資格情報アクセスおよび横移動の手法  
- Sysmon、Sigma、EQL を用いた検知戦略  
- 脅威インテリジェンスのエンリッチメントと相関分析  
- SOAR とプレイブックを用いた自動化  
- インシデントレスポンスおよび封じ込めワークフロー  
- 判断支援・チューニングのための LLM 連携  
- Active Directory ラボ環境および設定不備

---

## 目的

本プロジェクトの目的は以下のとおりです。

- 実践的な Purple Teaming ワークフローを提示する  
- 攻撃者の挙動を再現し、防御側の理解を深める  
- 現実的なシナリオに基づいた検知およびハードニング手法を提供する  
- SOC 運用、IR トレーニング、ツール検証のためのリファレンスラボとして活用できるようにする  
- インシデントレスポンスにおける自動化および AI 支援判断の可能性を探る
